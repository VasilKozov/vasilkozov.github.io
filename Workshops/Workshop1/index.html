AI + Creative Writing + Web Design Workshop

Create an AI-assisted story showcase website

Use a large language model to plan and write an original story, use image generation tools to create characters and locations, then build and publish a small multi-page website that presents the final result.

Final deliverable

  • ✅ Minimum 3000-word story
  • ✅ At least 3 story parts
  • ✅ At least 3 characters and 3 locations
  • ✅ At least 6 generated images
  • ✅ At least 3 linked HTML pages
  • ✅ Short presentation of your work

Workshop goal

The goal is not to let AI do everything automatically. Your task is to work with AI tools, make creative decisions, edit the result, organize it as a website, and explain your process.

Required language

Write the website, story, image descriptions, tables, presentation notes, and final explanation in English.

Suggested tools

ChatGPT or another LLM Microsoft Copilot Stable Diffusion WebUI Forge GitHub Pages quickstart

Step-by-step task

Follow the steps in order. You may use AI for ideas, drafting, image prompts, and coding support, but you must control the final creative and technical decisions.

1

Choose a genre and define the story direction

Open ChatGPT, Copilot, Gemini, a local LLM, or another text-generation tool. Choose a genre such as fantasy, science fiction, mystery, historical fiction, action, romance, alternate history, adventure, or a hybrid. Ask the AI to help you create a plan, but do not accept the first answer blindly.

2

Create at least three characters

Generate, discuss, and improve at least three characters. For each character, write down their name, role, visual appearance, personality, motivation, and any important object, power, profession, or conflict. Save the final visual description for image generation.

3

Create at least three locations

Create at least three different scenes or places where the story happens. They may be rooms, landscapes, planets, cities, vehicles, virtual worlds, microscopic environments, historical settings, or imaginary spaces. Each location must be used in the actual story.

4

Build the plot together with AI

Ask the AI for plot ideas, conflicts, turning points, and possible endings. Improve the result by asking for alternatives, stronger motivation, clearer conflict, better pacing, or a more surprising ending. All main characters and locations should have a purpose in the story.

5

Write and edit the full story

Generate a story of at least 3000 words. It must have at least three parts: beginning, middle, and end. Continue improving it until it is coherent, readable, and consistent with your characters and locations. You may add extra parts, an epilogue, or non-standard structure if the full story remains understandable.

6

Generate images

Create images for your characters and locations. You need at least six different generated images. You can use Copilot, ChatGPT image generation, Stable Diffusion, ForgeUI, Midjourney, or another tool. Keep notes about which tool you used and what prompt produced each important image.

7

Create at least three HTML pages

Build a small website using HTML and CSS. Each page should contain a different part of the story and should include relevant images. The first page should introduce the story; the last page should finish it. You may use JavaScript, Bootstrap, Tailwind, or your own CSS if it improves the result.

8

Add navigation

Each page must allow readers to move through the story. Add clear navigation such as Previous / Next buttons, a menu, chapter cards, or story arrows. Test every link before submitting.

9

Publish or prepare the project folder

If you publish online, GitHub Pages is recommended. Create a repository, upload your HTML, CSS, JavaScript, and image files, then enable GitHub Pages from the repository settings. GitHub notes that publication may take up to 10 minutes after changes are pushed.

10

Present your project

Present the final website, explain your story idea, show the generated images, and describe how you used AI. Focus on your decisions: what worked, what failed, what you changed, and what you learned.

Starter prompt

Use this as a starting point and replace the parts in brackets.

Hello, my task is to work with you to create a short story of at least 3000 words. The story must have three main parts: beginning, middle, and end.

The genre I want is [genre]. The tone should be [tone: mysterious / funny / dramatic / adventurous / educational].

First, help me define three interesting characters with names, visual descriptions, motivations, and roles in the plot. Then help me define three locations that will be used in the beginning, middle, and end of the story.

After that, help me build the plot, write the story, create image-generation prompts, and plan a small HTML website where I will present the story.

Prompting strategy

Ask for alternatives. Do not accept only one idea. Ask for three versions, compare them, and choose the best.

Control the structure. Tell the AI exactly what you need: number of characters, number of scenes, word count, tone, and constraints.

Improve in cycles. Ask the AI to rewrite weak parts, fix contradictions, make the ending stronger, or make descriptions more visual.

Keep notes. Save the final descriptions, prompts, and tools used. You will need them for your tables and presentation.

Useful prompt templates

Character refinement

Improve this character so they have a clear goal, weakness, visual identity, and connection to the main conflict.

Location design

Describe this location visually so it can be used both in the story and as an image-generation prompt.

Plot repair

Find problems in this plot and suggest fixes for pacing, motivation, conflict, and ending.

Image prompt

Turn this character/location description into a detailed image-generation prompt with style, lighting, composition, and mood.

HTML planning

Plan a three-page HTML website for this story. Suggest page titles, image placement, navigation, and CSS style.

Presentation reflection

Help me explain how I used AI, what I changed manually, and what difficulties I had during the project.

Stable Diffusion / ForgeUI installation help

This section is optional. Students can use any image-generation tool. Use local Stable Diffusion only if your computer is suitable and you have enough time to set it up.

Recommended option: Stable Diffusion WebUI Forge one-click package ⌄

Forge is a fork/platform based on Stable Diffusion WebUI that focuses on easier development, resource management, inference speed, and experimental features.

  1. Open the official Stable Diffusion WebUI Forge GitHub page.
  2. Download the one-click installation package recommended in the Forge README.
  3. Extract the archive to a folder with a simple path, for example D:\AI\Forge.
  4. Run update.bat first.
  5. Run run.bat to start Forge.
  6. Wait until the command window gives you a local web address, usually similar to http://127.0.0.1:7860.
  7. Open the address in your browser and start generating images.

Important: The Forge README specifically recommends running update.bat because otherwise you may use an older version with bugs that have already been fixed.

Manual installation with Git and Python ⌄

Use this only if you are comfortable with command-line tools.

  1. Install Git.
  2. Install Python for Windows. During installation, enable the option to add Python to PATH if available.
  3. Open a terminal in the folder where you want to install Forge.
  4. Run: git clone https://github.com/lllyasviel/stable-diffusion-webui-forge.git
  5. Open the new folder and run webui-user.bat.
  6. After the first setup finishes, open the local address shown in the console.
Where to place models and images ⌄
  • Stable Diffusion checkpoint files usually go in models/Stable-diffusion.
  • LoRA files usually go in models/Lora.
  • Generated images are usually saved in an outputs folder.
  • For the workshop website, copy only the final selected images into your project folder, preferably using short English filenames such as elena.png or city-background.jpg.
If local installation is too difficult ⌄

You may use browser-based image tools instead. The important part of the workshop is that you create suitable visual material, keep track of your prompts/tools, and use the images in your final website.

Evaluation rubric

This rubric follows the uploaded Excel evaluation sheet. The main score is out of 120 points. Content quality is also marked separately from 2 to 6.

Content grade

The story content receives a separate grade from 2 to 6. This reflects coherence, creativity, narrative quality, and how well the story uses the required characters and places.

Presentation score

The presentation and delivery criteria are scored with points, including timing, balance, synchronization, storytelling, formatting, and answers to questions.

Bonuses

Creativity and English use can add extra points. Use them to show originality, clear communication, and effort beyond the minimum.
